# 项目优化实施计划

> **版本:** v2.4
> **创建日期:** 2026-06-01
> **目标:** 按照项目分析结果执行 M1+M2 阶段优化

***

## 阶段目标

基于《项目分析与改进计划》文档，本次优化聚焦于：

1. **D3 按需导入优化** - 减小打包体积 15-20%
2. **图页面统一使用 Visualizer 组件** - 消除自定义 SVG 渲染不一致
3. **提取重复布局计算逻辑** - 减少代码重复
4. **配置 Vitest 测试框架** - 建立自动化测试基础
5. **核心 Hooks 单元测试** - 覆盖率 > 70%
6. **React.memo 性能优化** - 减少不必要重渲染

***

**目标：** 将 `import { select } from 'd3'` 改为按需导入子模块

**文件清单：**

1. `src/utils/d3Imports.js` - 创建统一的 D3 按需导入导出
2. `src/visualizers/arrayVisualizer.js` - 更新导入
3. `src/visualizers/stackVisualizer.js` - 更新导入
4. `src/visualizers/queueVisualizer.js` - 更新导入
5. `src/visualizers/linkedListVisualizer.js` - 更新导入
6. `src/visualizers/treeVisualizer.js` - 更新导入
7. `src/visualizers/graphVisualizer.js` - 更新导入
8. `src/visualizers/sortVisualizer.js` - 更新导入

**实施细节：**

* 分析每个 visualizer 使用的 D3 功能（select, transition, drag, forceSimulation 等）

* 在 `d3Imports.js` 中统一导出

* 替换所有 `from 'd3'` 为 `from '../utils/d3Imports'`

### 第二步：图页面统一使用 Visualizer 组件

**实施细节：**

## 风险与注意事项

1. **D3 按需导入**：需确保所有使用的 D3 功能都被正确导出
2. **图页面重构**：需仔细测试 force 模拟与拖拽交互
3. **测试框架**：首次配置可能需要调试环境问题
4. **React.memo**：需避免过度优化导致 props 更新失效
