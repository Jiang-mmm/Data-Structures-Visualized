| 排序可视化 | 30-100 | 30-45 fps | ⚠️ 需优化 |
| 图力导向 | ≤ 10 | 60 fps | ✅ 优秀 |
| 图力导向 | 10-20 | 40-50 fps | ⚠️ 需优化 |
| 图力导向 | > 20 | 15-25 fps | 🔴 需优化 |

### 3.2 性能瓶颈定位

**瓶颈 1: D3 全量渲染模式**
- 位置: 所有 `render*` 函数
- 原因: `selectAll('*').remove()` 后重新创建所有 DOM
- 影响: 大数据量时 DOM 操作成本高
- 建议: 引入 D3 enter/update/exit 增量更新（在关键路径）

**瓶颈 2: 图力导向模拟**
- 位置: `graphVisualizer.js` → `forceSimulation`
- 原因: 模拟计算在主线程执行
- 影响: 节点>20 时阻塞渲染
- 建议: 迁移到 Web Worker 或降低模拟频率

**瓶颈 3: ResizeObserver 防抖**
- 位置: `useVisualizer.js`
- 原因: 100ms 防抖可能在快速窗口变化时多次触发
- 影响: 轻微性能损耗
- 建议: 增加防抖到 200ms 或使用 `requestAnimationFrame`
