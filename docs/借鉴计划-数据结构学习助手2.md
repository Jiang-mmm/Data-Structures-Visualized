## 3. 技术适配方案

### 3.1 语言与类型迁移

- 项目2 为 JavaScript（`.js`/`.jsx`），项目3 为 TypeScript strict。所有借鉴代码必须重写为 `.ts`/`.tsx`，补齐类型声明。
- 优先复用项目3 的 `src/types/*.d.ts` 与 `hooks.d.ts`，避免新增第二套类型体系。
- 字符串算法/高级数据结构可先从项目2 移植核心逻辑到 `src/algorithms/` 的 `.ts` 文件，再补可视化。

### 3.2 状态管理融入

- 项目3 已形成 `useDataStructureState` 基座：持久化、日志、动画锁、撤销/重做。新增数据结构必须继承该基座，不能引入项目2 的 `useUndoRedo` 造成双历史栈。
- 全局交互组件（搜索、设置、成就、笔记）可复用项目2 的 `Context` 思想，但应收敛到现有 `useGlobalSettings` 或新建轻量 Context，禁止每个功能一个 Context。
- 学习相关数据（Quiz、Review、Achievements）建议接入 `useLearningProgress` 的 CustomEvent 同步机制，确保 `ProgressOverview` 实时更新。

### 3.3 可视化层适配

- 项目3 采用 **全清+全绘** 策略，项目2 采用 **D3 data join + action 驱动**。借鉴时不应推翻现有策略。
- 动画高亮逻辑可直接吸收项目2 的 `action` 语义（如 `compare`/`swap`/`visit`），但实现仍走 `animationEngine.ts` 的 `duration`/`wait`/`transition` API。
- 新增高级数据结构的可视化器需遵循项目3 规范：
  - 从 `src/utils/d3Imports.ts` 导入 D3；
  - 使用 `viewBox` + `className="w-full h-full"`；
  - 定义 `LARGE_DATA_THRESHOLD` 并跳过动画；
  - 每步检查 `anim?.isAborted?.()`；
  - SVG 元素设置 `role`/`aria-label`。

### 3.4 动画与性能工具接入

- `rafThrottle` 可放入 `src/utils/animationFrameThrottle.ts`（或合并到 `debounce.ts`），在 `useVisualizer` 的 ResizeObserver 回调、`graphVisualizer` 的 tick 事件中使用。
- Web Worker 排序：在 `src/workers/sortWorker.ts` 中实现，用 `sortWorkerClient.ts` 封装 Promise 接口。注意项目3 的排序算法接口为 `SortAlgorithm.execute(arr, animationFns, ...)`，需将“生成 steps”与“执行动画”分离：Worker 只负责生成 steps，主线程调用 `animationEngine` 播放。
- 虚拟滚动：优先用于 `LogPanel` 与 `Timeline`，避免破坏键盘导航与屏幕阅读器体验。

### 3.5 国际化与文案

- 所有新增 UI 文案必须走 `src/i18n/useI18n.ts` 的 `t()`，先补齐中文，英文可后续补全。
- 题库 `quizData`、成就 `achievements`、应用场景 `applicationScenarios` 需拆分为 `src/data/` 下的 TS 文件，并支持 `t()` 键。

### 3.6 测试策略

- 新增算法必须有单元测试（参考项目2 的 `tests/` 用例）。
- 新增组件/Hook 必须补充到 `src/__tests__/`。
- 每阶段完成后执行：`npm run lint && npm run typecheck && npm run test:run && npm run build`，确保 0 错误。

## 4. 实施路线（建议分 4 个阶段）

### 阶段一：全局交互与教学辅助层（第 1-3 周）

1. **GlobalSearch** 全局搜索：支持中英文、快捷键、页面跳转。
2. **SettingsPanel** 统一设置面板：字体、代码主题、动画预设、复杂度标签、紧凑模式。
3. **GuideTooltip / GuidedTutorial** 新手引导：首页 + 5 个核心数据结构页面试点。
4. **ExportMenu** 导出增强：SVG → PNG、日志导出、代码片段导出。
5. **PageSkeleton** 懒加载骨架屏（可选）。

**阶段交付标准：**
- 4 个 P1 特性全部可用；
- 新增组件测试覆盖率 ≥ 60%；
- `npm run lint && npm run typecheck && npm run test:run && npm run build` 全通过；
- 文档更新：`docs/借鉴计划-数据结构学习助手2.md` 勾选已完成项。

### 阶段二：学习闭环与测验成就（第 4-7 周）

1. **Quiz** 测验系统：选择/判断题，按数据结构分类，结果写入学习进度。
2. **Achievements** 成就系统：访问、动画、测验、收藏等 8-10 类成就。
3. **ApplicationScenarios** 应用场景组件：为 5 个核心页面补充工业案例。
4. **Notes & Bookmark** 页面级笔记与收藏。
5. **ReviewReminder** 间隔重复复习（依赖 Quiz 完成）。

**阶段交付标准：**
- 学习进度可展示“已完成测验/成就/收藏”维度；
- `useLearningProgress` 同步机制正常；
- 新增 `src/data/quizData.ts`、`src/data/achievements.ts` 等数据文件。

### 阶段三：高级数据结构与算法扩展（第 8-16 周）

按“8 步新增流程”逐个落地：

1. **P1 优先（快速教学价值高）：**
   - AVL 树、红黑树、跳表、并查集。
2. **P2 高级结构（逐步补充）：**
   - B 树、线段树、树状数组、稀疏表、布隆过滤器、LRU Cache。
3. **字符串算法：**
   - KMP、Manacher、Rabin-Karp（每个 4-5 天）。
4. **图算法补充：**
   - A* 寻路、关键路径法 CPM。

**阶段交付标准：**
- 每新增一种数据结构：Hook + Visualizer + Page + 路由 + 导航 + 首页卡片 + 单元测试 + 学习配置；
- 所有页面遵循项目3 的 `useDataStructureState` 基座与 `animationEngine` 动画规范；
- 每完成 4 个页面做一次全量 E2E 回归。

## 6. 风险识别与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 从 JS 迁移到 TS 引入类型错误 | 中 | 分文件迁移，每文件补齐类型后立即跑 `npm run typecheck` |
| 新增高级数据结构动画复杂（红黑树/B树） | 高 | 先实现核心操作与静态渲染，动画作为第二阶段；复用项目2 的旋转/分裂逻辑 |
| 学习辅助功能过多导致 Context 膨胀 | 中 | 统一收敛到 `useGlobalSettings` 或轻量 Store，禁止每个功能独立 Context |
| Web Worker 与现有排序接口不兼容 | 中 | Worker 只负责生成 steps，主线程保留 `animationEngine` 播放控制 |
| Bundle 超预算 | 中 | 新增页面必须 `React.lazy`；高级结构按页面拆 chunk；定期跑 `npm run build:analyze` |
| 无障碍（a11y）回归 | 中 | 每阶段跑 `node e2e/test-a11y.js`，新增组件必须补充 `aria-label`/`role` |
| 文档与测试滞后 | 高 | 严格执行“新增必测试、变更必文档”，每阶段更新本计划与 `PROJECT_SUMMARY.md` |
