## 7. 关键发现

### 7.1 规则升级决策（Q4 拍板：M4-3 收尾时升级为 `error`）

**事实**：v17 GA 现有 17 P0+P1 + 3 P2 目标已 100% `t()` 化（569 个 `t()` 调用），但 v19 范围**外**的以下位置仍存在少量硬编码中文（按规则 §16.1 `design-md/` 默认禁读不适用；按 v15.x 之前的累积遗留）：

| 位置 | 数量 | 性质 |
|------|------|------|
| `src/hooks/*` 内部日志 | ~30 处 | 开发者向日志，非用户可见 |
| `src/configs/learning/step.*` 教学文案 | ~70 处 | 属于 M7 范围，未启动 |
| `src/utils/*` 错误消息 | ~15 处 | 属于 M6 范围，未启动 |
| `src/components/*` 组件 props 默认值 | ~10 处 | 属于 M5 范围，未启动 |

**决策**：**保持 `no-hardcoded-chinese-in-jsx` warn 级**。原因：① 升级为 `error` 会立即 fail（v19 范围外仍有 ~125 处硬编码）；② M5+M6+M7 阶段完成后再升级；③ 当前 warn 级别 + v19 范围 100% `t()` 化已足够防止新增。

### 7.2 聚合层状态

`src/i18n/locales/index.ts` 已于 M2 阶段交付（导出 types / integrity / pseudoLocale + zh/en 顶层聚合）。**M4 阶段无需修改聚合层**（20 目标全部使用现有 `useI18n().t()` API）。
